# Common Parts Access — Homepage, Tone, and Product Positioning

Phase 1: Open Platform & Database Formation
0. Purpose of This Document

This document defines the exact positioning, tone, structure, and design principles for the Common Parts Access homepage and interface.

It is intended for:

* Developers
* Designers
* Product owners
* Content writers

It ensures Common Parts Access presents itself accurately, credibly, and sustainably.

This is not a marketing document.
This is a product and institutional positioning reference.

1. Strategic Reality: Current Phase vs Long-Term Vision
What exists today

Common Parts Access is:

An open platform for publishing and accessing digital spare parts
A structured and growing library of repair models
A platform operated by the Common Parts project
A tool for contributors to publish useful repair components

It is NOT yet:

A regulatory body

A public institution

A certification authority

A brand-integrated distribution network

A secure streaming protocol system (CPSP not yet implemented)

All messaging must reflect this honestly.

What we are building toward (but must not pretend already exists)

Common Parts aims to develop:

A standardized framework for digital spare parts

Secure print authorization protocols (CPSP)

Manufacturer integration

Certified part distribution

These must be presented as roadmap goals, not current features.

2. Core Positioning of Common Parts Access

Canonical definition:

Common Parts Access is an open platform for publishing and accessing digital spare parts for everyday object repair, and the first implementation aligned with the emerging Common Parts framework.

Short version:

An open platform for digital spare parts.

Do NOT describe it as:

“Public infrastructure”

“Official repair authority”

“Certification system”

Those would be inaccurate at this stage.

3. Tone and Voice Requirements
Tone must be:

Calm

Neutral

Clear

Precise

Honest

Functional

Tone must NOT be:

Gamified

Playful

Thematic (no harbor, docking, nautical metaphors)

Overhyped

Activist or ideological

Startup hype language

Avoid phrases like:

“Join the revolution”

“Dock your models”

“Earn rewards”

“Become a repair hero”

We are building structured infrastructure, not entertainment.

4. Homepage Structure (Exact Layout)

This defines the homepage from top to bottom.

SECTION 1 — Header

Left:

Common Parts Access

Right navigation:

Browse Parts

Publish a Part

About

Common Parts

Keep navigation minimal.

SECTION 2 — Hero Section
Headline

Recommended:

Repair starts with access to the right part.

Alternative acceptable:

A shared platform for digital spare parts.

Avoid grand institutional claims.

Subheading

Recommended:

Common Parts Access is an open platform for publishing and accessing digital spare parts, helping extend the life of everyday objects.

This must remain truthful and concrete.

Primary Button

Browse Parts

Secondary Button

Publish a Part

These are the two core actions.

SECTION 3 — Explanation Section

Title:

What is Common Parts Access?

Body:

Explain clearly and simply:

It is a platform where spare parts can be published and accessed

It helps make repair easier and more accessible

It is part of the broader Common Parts project, which aims to standardize digital spare parts over time

Example wording:

Common Parts Access provides a structured platform where repair parts can be published, discovered, and used. It is designed to support the growing ecosystem of digital manufacturing and repair.

No exaggerated claims.

SECTION 4 — Contributor Call (Critical Section)

This is essential for building the database.

Title:

Publish repair parts

Body:

Explain contributor role clearly and seriously.

Example wording:

We are building a structured library of repair parts. Contributors can publish models that help others repair everyday objects. Early contributions help establish a reliable and accessible foundation for digital spare parts.

CTA Button:

Publish a Part

Important:

This must feel like contributing to useful infrastructure — not like joining a game.

SECTION 5 — How It Works

Simple 4-step explanation:

A part is published to the platform

Others can discover and download it

The part can be manufactured using local or personal 3D printing

The object is repaired and continues to be used

Keep visuals minimal and diagrammatic.

Avoid cartoons or playful illustrations.

SECTION 6 — About Common Parts

Title:

About Common Parts

Body example:

Common Parts is an ongoing project focused on structuring and standardizing digital spare parts. Common Parts Access is the first platform built within this framework.

Link to:

commonparts.org (or placeholder if not yet live)

This creates legitimacy without exaggeration.

SECTION 7 — Footer

Include:

About

Publish a Part

Contact

Common Parts

Optional future:

Governance

Documentation

Footer must be minimal and calm.
